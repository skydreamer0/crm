# AI 安全與開發規範 (SECURITY GUIDELINES)

## 絕對禁止事項 (CRITICAL: NEVER DO THIS)
1. **禁止明文傳遞憑證**：絕對不允許在任何形式的對話、提示詞、子代理 (Subagent) 任務描述中，直接揭露或寫出使用者的帳號、密碼 (例如 `SYNMOSA-DC\...`) 或任何敏感資訊。
2. **禁止追蹤敏感檔案**：`.env`、金鑰檔案或含有個資的原始 Excel/CSV 檔案，絕對不允許被版本控制系統 (Git) 追蹤。

## 開發與執行規範
1. **憑證讀取唯一途徑**：所有自動化程式或爬蟲，**必須且只能**透過環境變數 (Environment Variables) 或 `.env` 檔案在本地執行時由作業系統或 Python 程式層級讀取。
2. **AI 工具使用限制**：若需要指派瀏覽器子代理 (Browser Subagent) 進行頁面分析，**嚴禁**將帳號密碼寫入 `Task` 描述中。如果需要登入後的頁面資訊，應請使用者手動提供 HTML，或者撰寫本地 Python 腳本將網頁結構儲存為純文字檔後由主要 AI 讀取。
3. **Log 記錄安全**：不得在任何公開或系統紀錄檔 (`automation.log`、終端機輸出體) 中打印出完整的密碼或敏感個資。

> **AI 系統備忘錄**: 本文件為最高層級的系統記憶與強制約束，在任何時候存取此專案時均自動生效。
